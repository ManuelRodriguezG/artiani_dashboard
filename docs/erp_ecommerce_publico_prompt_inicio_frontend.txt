PROMPT DE ARRANQUE - FRONTEND ECOMMERCE PUBLICO MASCOTAS

Proyecto ERP fuente:
C:\xampp\htdocs\panel_de_control

Objetivo:
Crear un frontend publico separado del ERP para un catalogo vivo de productos para mascotas. Fase 1 no es checkout: es catalogo, filtros, buscador, carrito local tipo cotizacion, validacion dry-run contra ERP y envio por WhatsApp.

Estado actual ERP:
- senal_frontend=amarillo_mock_contratos
- puede_iniciar_frontend_mock=true
- puede_integrar_datos_reales=false

Se puede iniciar:
- estructura del proyecto frontend;
- AGENTS.md;
- cliente API;
- mocks/fixtures;
- pantallas responsive;
- estados vacios;
- carrito local;
- flujo WhatsApp preparado.

No se debe integrar como datos reales hasta que el ERP devuelva:
- green gate ok=true;
- catalogo.tiene_item_real=true;
- cotizacion_dryrun.ok=true.
